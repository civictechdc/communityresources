DC Community Resources Data: The Solution.

Through a cross-sector, multi-stakeholder collaboration, the DC Community Resource Data working group has merged multiple local service directories into one comprehensive database that includes all of the non-profit agencies and public services in the District of Columbia. (This database is hosted on the Open Data DC catalog -- you can access it here.)

Rather than trying to build the ‘Right Application’ to serve this directory, we believe that a truly effective solution must instead establish this service data as a commons—a cooperatively shared resource, accessible by an ecosystem of community stakeholders and applications.

We can do this by making the directory openly available via an Application Programming Interface (API) that enables external programs to access the data. This will make it much easier for anyone to develop a directory service specifically designed for their community’s particular needs. By integrating the data commons into an ecology of stakeholders, we can also distribute the challenge of keeping the data up to date.

A Path Forward.

1) Establish a common data pool by merging multiple local databases into a single consolidated file in which each organization is assigned a unique ID. We have already accomplished this step! The data is available here.

2) Apply an “open” taxonomy to organize the common data pool. Currently, our consolidated directory lacks an established order for its entries (almost all information-and-referral systems currently use the 211/AIRS taxonomy, which is proprietary, so it cannot be used in an open environment). We may be able to use OpenEligibility.org, an open source social service taxonomy that has just recently been published. If deemed viable, this open taxonomy would be a common standard through which different applications could share the same "map" of the local service universe.

3) Establish cooperative agreements for the sustained viability of the common data pool. This step could potentially entail the formation of a “data coop”: a membership organization offering premium services to all local entities for a fee or a commitment of time for data management. See this recent example of a "data coop" organization.

4) Demo through a variety of applications, such as the mobile Open211 project; service-oriented software for case managers and social workers like Aunt Bertha; a LocalWiki that serves the structured data alongside unstructured space for users to contribute tacit knowledge; etc.

5) Engage users in participation, through an accompanying set of social programs, anchored in various community spaces and designed for various levels of skills and purposes. For instance: resource mapping projects in social service agencies and civic associations; digital literacy training through "wiki parties" at libraries; and “hackathons" that can bring users, developers, and social workers together to build tools that access the common data pool.

By sharing an open, common directory of DC’s community resources, we can greatly improve access to critical services; in the long run, we can enhance our collective capacity to understand and make decisions about how resources are allocated in our communities.

How to get involved.

The DC Community Resource Data working group is an ad hoc table of people and organizations interested in solving this problem together. You should join us! We are looking for community leaders, health and social workers, data geeks, and anyone who wants to learn about how these things work and how we can make them better.
